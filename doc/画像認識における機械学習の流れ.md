# 画像認識における機械学習の流れ

## 概要

画像認識AIは、画像データから特徴を学習し、対象物を検出・分類する技術です。

機械学習を利用した画像認識開発は、以下の流れで進めます。

```
データ収集
    ↓
アノテーション（正解付け）
    ↓
データ分割
    ↓
モデル選択
    ↓
学習（Training）
    ↓
評価（Validation / Test）
    ↓
推論（Inference）
    ↓
改善・再学習
```

---

# 1. データ収集

## 目的

AIに学習させるための画像データを集めます。

AIは大量の画像を見ることで、対象物の特徴を学習します。

## 例：人検出の場合

以下のような画像を収集します。

- 正面を向いた人
- 横向きの人
- 後ろ姿の人
- 座っている人
- しゃがんでいる人
- 複数人いる状態
- 暗い環境
- 一部が隠れている状態
- 異なる距離や角度

実際に使用する環境に近いデータを集めることが重要です。

---

# 2. アノテーション（正解付け）

## 目的

AIに「画像内のどこに何があるか」を教えます。

収集した画像に対して、人間が正解データを作成します。

## 例：人検出

画像：

```
+----------------+
|                |
|    +------+    |
|    | 人   |    |
|    +------+    |
|                |
+----------------+
```

登録する情報：

```
class : person

x座標
y座標

幅
高さ
```

このような正解付きデータを

**教師データ（Training Data）**

と呼びます。

---

# 3. データ分割

## 目的

AIの学習用データと評価用データを分けます。

例：

|種類|割合|用途|
|-|-|-|
|学習データ|80%|AIを訓練する|
|検証データ|10%|学習途中の評価|
|テストデータ|10%|完成後の性能確認|

学習に使用した画像だけで評価すると、
AIが本当に理解しているか判断できないため、
未知の画像を使って性能を確認します。

---

# 4. モデル選択

## 目的

用途に適したAIモデルを選択します。

代表的な画像認識モデル：

|モデル|特徴|
|-|-|
|YOLO|高速な物体検出が可能|
|Faster R-CNN|高精度な物体検出|
|CNN|画像分類で広く利用|
|Transformer系|近年利用が増加|

## 人検出の場合

リアルタイム処理が必要な場合は、

- YOLO
- SSD

などの物体検出モデルがよく利用されます。

---

# 5. 学習（Training）

## 目的

AIに画像の特徴を覚えさせます。

大量の教師データをAIモデルへ入力し、
対象物の特徴を自動的に学習します。

AIが学習する特徴：

- 色
- 輪郭
- 形状
- 模様
- 周辺との関係

など。

## 人検出の場合

AIは、

```
頭部
 ↓
胴体
 ↓
手足
 ↓
全体の形状
```

などの特徴から、
「人らしさ」を判断する基準を作ります。

---

# 6. 評価（Validation / Test）

## 目的

学習したAIが正しく認識できるか確認します。

主な評価指標：

|項目|意味|
|-|-|
|Accuracy|全体の正解率|
|Precision|誤検出の少なさ|
|Recall|見逃しの少なさ|
|mAP|物体検出モデルの代表的な評価指標|

## 人検出の場合

重要なポイント：

- 人を見逃さないこと
- 物を人と誤認しすぎないこと

安全装置などでは、
特にRecall（検出率）が重要になります。

---

# 7. 推論（Inference）

## 目的

完成したAIモデルを実際の装置で使用します。

基本的な流れ：

```
カメラ
 ↓
画像取得
 ↓
AIモデル
 ↓
人検出
 ↓
位置情報取得
 ↓
制御へ出力
```

## 例：設備安全システム

```
カメラ映像
 ↓
AIが人を検出
 ↓
危険エリア侵入判定
 ↓
PLCへ信号出力
 ↓
設備停止
```

---

# 8. 改善・再学習

## 目的

現場環境に合わせてAIの精度を向上させます。

発生する問題例：

- 暗い場所で検出できない
- 人が一部隠れている
- 背景を人と誤認する
- 距離によって精度が低下する

改善の流れ：

```
問題発生
 ↓
失敗画像を収集
 ↓
アノテーション追加
 ↓
再学習
 ↓
モデル更新
```

このサイクルを繰り返して精度を向上させます。

---

# 画像認識AIシステム全体

```
[カメラ]
    ↓
[画像取得]
    ↓
[画像処理]
    ↓
[AIモデル]
    ↓
[認識結果]
    ↓
[座標・判定]
    ↓
[PLC / ロボット制御]
```

---

# まとめ

画像認識AIの開発は、

1. 画像を集める
2. 正解データを作成する
3. AIモデルを学習させる
4. 精度を評価する
5. 実際の装置で使用する
6. 問題を改善して再学習する

という流れで進めます。

画像認識AIの性能は、
AIモデルの種類だけではなく、

**実際の使用環境に近い良質な教師データをどれだけ準備できるか**

によって大きく左右されます。
