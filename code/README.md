## 📂 各ファイルの解析結果

### 1. `browser.py` (LAN内・Webブラウザ配信版)
Flaskを使用して、処理された映像をWebブラウザ上でリアルタイムストリーミングするサーバープログラムです。

* **主な処理内容**:
    * OpenCVの`StereoBM`を用いて本格的な視差（Disparity）マップと距離マップを生成。
    * YOLOv8で人物（Class 0）を検知。
    * 誤検知やノイズを減らすため、人物のバウンディングボックスの**中央部分（幅35〜65%、高さ20〜80%）の領域**のみを距離計測に使用。さらに、その領域内の距離データの**30パーセンタイル値**を採用することで、安定した計測を実現。
    * 人物が0.70m以内に近づくと、該当ピクセル領域を赤く半透明で塗りつぶし。
    * `Flask`のマルチパート配信(`multipart/x-mixed-replace`)を使い、ポート5000番でMJPEG形式の映像を配信。

### 2. `sensor_camera_calibration.py` (ローカルGUI・詳細描画版)
`browser.py`とほぼ同じ画像処理エンジンを使用していますが、Web配信ではなくローカルのディスプレイに直接OpenCVのウィンドウを表示するためのプログラムです。

* **主な処理内容**:
    * 事前キャリブレーションファイル(`stereo_camera_calibration.npz`)を読み込み、レンズ歪みや平行化の補正(`cv2.remap`)を実行。
    * 距離の算出方法が`browser.py`と少し異なり、人物の枠内全体の距離データのうち**最小値（一番近い距離）**を採用。
    * 描画テキストの視認性を高めるため、黒い縁取り（影）の文字の上に白い文字を描画する工夫を追加。

### 3. `warning.py` (軽量・テンプレートマッチング版)
一番初めのプログラム。`StereoBM`による画面全体の重い距離計算を行わず、簡易的かつ高速に特定の人物との距離を測るスクリプトです。

* **主な処理内容**:
    * YOLOで左カメラの人物を検知し、その中心座標の小さな矩形（20x20ピクセル）を切り出し。
    * 切り出した画像を右カメラの映像から`cv2.matchTemplate`で探索（テンプレートマッチング）し、一致したズレ（視差）から距離を計算。
    * **安全装置（警告）機能**: 0.5m以内に人物が入った状態が3フレーム（`ALERT_THRESHOLD`）連続すると、画面全体を赤くオーバーレイ表示し「!!! WARNING !!!」という強い警告を出力。
    * *注意点*: このファイルのみ、ベースラインの数値が `0.114` に設定されています。カメラの物理的な幅を変更した際にテストされたものと推測されます。

### 4. `yolospace2.sh` (Docker起動用シェルスクリプト)
NVIDIA GPUを利用してコンテナを起動するためのDockerのrunコマンド（bash/shellスクリプト）です。

* **主な処理内容**:
    * `--runtime nvidia` でGPUを有効化。
    * `--device /dev/video0 ~ video2` でホストマシンのカメラ3台分をコンテナにパススルー。
    * GUI表示（X11サーバー）をホストと共有し、Docker内からウィンドウを開けるように設定。
    * ローカルにあるキャリブレーションファイル（`stereo_camera_calibration.npz`）をコンテナ内のワークスペースにマウント。
